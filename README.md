### Вопросы к заданию

**Чем async/await лучше чистых .then()?**  
Async/await является синтаксическим сахаром, позволяет писать асинхронный код в синхроном стиле, что улучшает читаемость кода. Упрощает систему отладки кода.

**Почему fetch не падает в catch при ошибке 404?**  
Потому что сервер вернул ответ. Catch не срабатывает при http-ошибках. В данном случае статус 404 говорит нам об отсутствии данных. Для того чтобы сработал catch необходимо кинуть ошибку с помощью throw при обработке статуса ответа. 

**Зачем проверять Content-Type ответа?**  
Проверять content-type необходимо, чтобы клиент понял в каком формате пришли данные и правильно их интерпретировал. Для защиты от MIME-sniffing. 
Браузер автоматически пытается определить тип контента, что может привести к опасным скриптам.
Если ожидался JSON, а пришел HTML (например, страница ошибки 500), проверка заголовка поможет корректно обработать эту ситуацию, а не пытаться парсить HTML-код как JSON.
